Eindopdracht

De eindopdracht voor dit vak is om het spel Stratego te maken in je browser. Niet zomaar een stratego, maar eentje die je online tegen een computer kan spelen of tegen iemand anders uit jouw klas. We hebben daarvoor een API gemaakt waarmee je moet communiceren om het spel te spelen. De documentatie daarvan vind je op deze pagina

Je mag jouw versie zo mooi maken als je zelf wil, maar het moet wel aan onderstaande eisen voldoen. Begin niet te lang met wachten om aan de opdracht te beginnen, je hebt zeker een paar weken nodig om het goed af te maken!

Ter inspiratie kan je een kijkje nemen naar onze eigen uitwerking

Nog nooit Stratego gespeeld? Nox van Nox' Spellenzolder legt het voor je uit:

Organisatorische eisen

  • Omdat we de opdracht volgend jaar weer willen gebruiken mag je geen gebruik maken van een publieke repository om je code in op te slaan. Als je toch gebruik wil maken van GitHub kan je via deze link een private repository aanvragen

  • Je maak het project in duo's. Uitzonderingen moeten eerst worden overlegd met de practicumdocent.
  • Jullie moeten allebei alle code goed kunnen uitleggen. "Ik weet niet hoe dat werkt want dat heb ik niet gemaakt" levert je minpunten op.
  • Het project lever je in op BlackBoard (linkje komt binnenkort beschikbaar). Dit doe je voor de deadline van maandag 19 juni. Zodra de deadline verstreken is wordt er een mailtje rondgestuurd met de inschrijflink.
  • De assessments zullen worden gepland in week 9.

Functionele eisen:

  • Het spel doorloopt verschillende fases. Een paar fases zijn bijvoorbeeld: Spel uitzoeken, speelstukken plaatsen, mijn beurt om een speelstuk te verplaatsen, tegenstanders beurt, einde spel. Volg de flow van het spel zoals beschreven in de API documentatie. Elke fase heeft zijn eigen user interface en regels, je mag bijvoorbeeld niet een speelstuk verplaatsen als het de beurt van de tegenstander is.
  • Bij het uitzoeken van een spel kan de gebruiker kiezen uit 'Speel tegen een computer' of 'Speel tegen iemand anders'.
  • Het moet mogelijk zijn om een lijst van alle lopende games te zien, en een game te selecteren om mee verder te spelen.
  • Tijdens het spel kan de gebruiker alleen een speelstuk verplaatsen als hij aan de beurt is. Met een refresh knop kan de gebruiker zien of hij weer aan de beurt is als hij wacht op de tegenstander (polling mag ook maar wees aardig voor de server, niet 10x per seconde!).
  • Zorg ervoor dat de gebruiker geen zetten kan doen die niet mogen, controleer dit zonder de API te gebruiken. Hierbij hoef je lastige regels als 'niet 5x heen en weer bewegen' of bepalen of het gelijk-spel is niet mee te nemen.
  • Als het spel is afgelopen (gewonnen of verloren) laat je dat duidelijk aan de gebruiker zien.

Technische eisen

  • Alle code is geschreven in JavaScript, dus geen PHP of andere server-side talen.
  • Het hele spel wordt gespeeld op één enkele pagina, gebruik dus geen iframes of links naar andere pagina's
  • Plaats alle JavaScript code in losse JavaScript bestanden die je include in de HTML. Plaats geen JavaScript in de HTML zelf.
  • Je mag geen gebruik maken van JavaScript frameworks zoals Angular, React.js of Backbone. jQuery (& jQuery UI) is toegestaan. Overleg altijd even met je practicumdocent als je JavaScript libraries wilt gebruiken die specifieke functionaliteit makkelijker maken (zoals Three.js of SoundJS).
  • Zorg voor overzichtelijke leesbare JavaScript code. Code die geminified is kunnen we tijdens het assessment niet beoordelen.
  • Dump niet alle code in één groot bestand. Deel je code netjes in en verdeel dit over meerdere bestanden.
  • Maak gebruik van constructor functies, prototypes, of ES6 classes om je code op te delen in verschillende soorten objecten. Denk bijvoorbeeld aan: GameList, Game, Board, Piece en Square. Zorg ervoor dat minstens 80% van je code in de 'methodes' zitten van deze objecten (Dat is minder moeilijk dan het lijkt, in alle C# en Java die je tot nu toe hebt geschreven staat 100% van je code in methodes)
  • Gebruik onze API om online tegen iemand anders Stratego te kunnen spelen. Alle API aanroepen maak je doormiddel van AJAX requests. Je mag hier jQuery voor gebruiken.

Extra eisen

Heb je alle functionele eisen goed geimplementeerd? Dan heb je een 7, je kan dan nog 3 bonuspunten verdienen door extra eisen te implementeren:

  • Maak het spel extra mooi met animaties en geluidjes.
  • Maak een drag & drop interface voor het plaatsen van de startopstelling.
  • Maak gebruik van de websockets API
  • Laat de historie van het spel zien
  • Deduceer aan de hand van de historie van een potje waar de speelstukken van de tegenstander staan waarvan je het type al weet, en laat dat zien aan de gebruiker.
  • Maak een goede scheiding tussen Model, View en Controller code
  • Voor de absolute 10 moet je zelf nog iets verzinnen. Misschien een knop die automatisch een slimme zet suggereert? Of een mogelijkheid om de layout van het bord op te slaan? Verras ons!
